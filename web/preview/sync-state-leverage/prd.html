Core Addition

Context Manager

Key addition: The dropout_context() method turns enter/exit into a context manager. This is the mechanism that makes "forgot to restore" impossible.
@contextmanager def dropout_context(self): """Context manager for dropout application.""" self.enter_dropout() try: yield except Exception: self.exit_dropout(success=False) raise else: self.exit_dropout(success=True)

Data Flow

Trainer owns SyncState │ ├─→ WeightBus(strategy, state) │ └─→ sync() uses state.initialized, state.complete_sync() │ ├─→ WeightLoader(..., state) │ └─→ sync_lora_direct() uses state.cache_stale, state.begin_sync() │ └─→ apply_lora_dropout(model, rate, state) └─→ uses state.enter_dropout(), state.exit_dropout()

All state flows from Trainer. No module owns its own flags. State mutations happen through SyncState methods, which handle locking internally.

Emergence

What Surprised Us

Surprise

SyncState was already implemented. The leverage point isn't "design a state machine" — it's "wire the one that exists." The sync_state.py file has all the right methods. It's just dead code.

Surprise

Context managers are the real unlock. The methods exist but dropout_context() doesn't. Adding it makes the restore-forget bug class structurally impossible — not caught at runtime, not possible to write.

Surprise

This deletes code. Net line count goes down. The scattered flags, their associated locks, and their synchronization logic all get removed. Fewer lines, fewer bugs, simpler mental model.
