Executive Summary

The Leverage Point

Root Cause: Every component (trainer, dataloader, advantage estimator, weight loader) manages its own mutable state scattered across __init__ fields. Checkpointing requires manually transcribing each field into state_dict() and back. The three locations (__init__, state_dict, checkpoint save/load) are always out of sync.

The fix: declare all mutable state in @dataclass StateSpec containers. Each component holds a single _state: ComponentState slot. Checkpoints serialize/deserialize the StateSpec directly via dataclasses.asdict(). Adding a new field without updating checkpoint code causes a TypeError at load time, not a silent production bug.

What Changes

4 new StateSpec dataclasses in types.py: TrainerState, DataLoaderState, AdvantageEstimatorState, WeightLoaderState. One CheckpointState container holds all of them. Checkpoint save/load becomes trivial: serialize the container, deserialize with validation.

What Disappears

17+ bug classes from the harden audit: TL-03, TL-11, DP-02, DP-06, TL-R2-02, TL-R2-04, AE-R2-01, AE-R3-01, AE-R3-04, DP-R3-01, TL-R3-04, TL-R3-07, WS-R3-03, WS-R3-04, and all future "state field not saved/restored" bugs.
